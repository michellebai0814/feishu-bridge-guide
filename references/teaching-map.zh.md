# 飞书 Bridge 教学地图

## 用途

这份文件给 agent（智能代理）读，不是给普通用户先看的首页。

作用只有一个：

- 当用户搞不清楚“这个功能到底该去哪里找”时，先帮他分对类。

## 五类理解法

这 5 类是教学用分类，不是官方唯一架构。

| 类别 | 大白话解释 | 常见内容 |
| --- | --- | --- |
| 飞书聊天命令 | 你在聊天里直接发一句命令 | `/help`、`/new`、`/provider`、`/dir` |
| 飞书后台权限/事件/回调 | 飞书应用后台里的设置 | 消息权限、`im.message.receive_v1`、`card.action.trigger` |
| 本地配置文件 | 电脑里的功能开关 | `resolve_mentions`、`tts`、`attachment_send`、`admin_from` |
| 本地项目目录 | agent 读取和输出文件的地方 | 教程、资料、图片、导出文件 |
| 网页管理后台 | 浏览器里的管理页面 | `/web setup` 打开后的页面 |

## 词语翻译表

| 原词 | 给普通人的解释 |
| --- | --- |
| command | 聊天里发的口令（命令） |
| permission | 飞书允不允许这个应用做事 |
| event | 飞书把“发生了什么”通知给你 |
| callback | 飞书把“用户点了什么”通知给你 |
| config.toml | 本地配置文件 |
| working directory | 当前读取资料的文件夹 |

## 常见路由

| 用户问题 | 先带去哪里 |
| --- | --- |
| `/help`、`/new`、`/switch` 怎么用 | 飞书聊天命令 |
| 为什么收不到消息 | 飞书后台权限或事件 |
| 为什么卡片按钮没反应 | 飞书后台回调 |
| 为什么不能自动艾特 | 本地配置文件，再补看飞书后台权限 |
| 为什么发不回图片或 PDF | 本地配置文件，再补看本地项目目录 |
| 为什么读不到我的教程 | 本地项目目录 |
| 怎么打开网页管理页 | 先发 `/web setup` |

## 回答模板

推荐先用这句开头：

```text
这个问题大概率不在一个地方全解决，它更像属于：聊天命令 / 飞书后台 / 本地配置 / 本地资料 / 网页管理 其中之一。
我先帮你判断最应该先看哪一类。
```

然后只给：

1. 当前最可能的类别
2. 这个词的大白话解释
3. 下一步先去哪里看

不要一上来丢一整篇技术手册。
