# 飞书 Bridge 快速开始

这份是最短版本。

如果你不想先看长文，先看这一页就够了。

## 先说结论

先不要问“为什么不工作”，先问：

`这个事情属于聊天命令、后台权限、本地配置、本地资料，还是网页管理？`

只要先分对类，排查会快很多。

## 5 步跑通

1. 把 bot 拉进飞书群或私聊它。
2. 发 `/help`，看它有没有正常回应。
3. 发 `/new`，新建一个会话。
4. 直接发一句人话测试，例如：

```text
帮我把飞书 bot 配置教程拆成 3 步，新手也能看懂。
```

5. 如果某个功能不工作，用下面这张表先判断去哪里看。

## 出问题先看哪里

| 现象 | 先去哪里看 |
| --- | --- |
| 命令没反应 | 飞书聊天页 |
| 发不了消息、读不了群信息 | 飞书开放平台后台权限 |
| 收不到消息事件 | 飞书开放平台后台事件 |
| 卡片按钮点了没反应 | 飞书开放平台后台回调 |
| 自动艾特、语音、文件回传不生效 | 本地配置文件 |
| 读不到你的教程或本地资料 | 本地项目目录 |
| 想打开 Web 页面 | 先发 `/web setup` |

## 最常见的几个词

- `口令（命令）`：你在聊天里直接发的斜杠命令，比如 `/help`
- `权限`：飞书允不允许应用做某件事
- `事件`：飞书把“发生了什么”通知给你
- `回调`：飞书把“你点了什么按钮”通知给你
- `配置项`：你本地开的功能开关

## 一句通用提问模板

你可以把这段话直接发给 agent：

```text
先别急着给我一堆步骤。
先判断我这个问题更像下面哪一类：
1. 飞书聊天命令
2. 飞书后台权限
3. 飞书后台事件或回调
4. 本地配置文件
5. 本地资料或工作目录
6. 网页管理后台

然后只告诉我下一步该去哪看。
```
